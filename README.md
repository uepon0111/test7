# 楽曲ライブラリ（PROJECT SEKAI 非公式楽曲データベース）

sekai-world/sekai-master-db-diff と sekai.best のアセットCDNから楽曲データを取得して表示する、
GitHub Pages 用の静的Webアプリです。ビルド不要（フレームワーク・バンドラなし）の
素のHTML / CSS / JavaScript（ESモジュール）で構成されています。

## デプロイ方法（GitHub Pages）

1. このzipを展開し、中身（`index.html` / `css/` / `js/` / `.nojekyll`）をリポジトリの
   直下（またはPages公開対象フォルダ）にそのまま配置してコミット・プッシュします。
2. GitHubリポジトリの **Settings → Pages** で、公開ブランチとフォルダ（例: `main` ブランチの `/root`）
   を指定します。
3. 数分後に発行されるURLでアクセスできます。サーバーサイドの処理は一切不要です。
4. ルーティングは `#/` `#/song/{id}` のハッシュ形式のみを使用しているため、
   404対策やリライト設定は不要です。

ローカルで確認する場合は、`index.html` をそのままブラウザで開くのではなく、
`python3 -m http.server` などの簡易HTTPサーバー経由で開いてください
（`fetch` はローカルの `file://` 直接読み込みでは正しく動作しないブラウザがあります）。

## ファイル構成（役割ごとに分割）

```
index.html          画面の骨格（各JS/CSSが差し込む土台のみ）

css/
  variables.css      配色・フォント・余白などのデザイントークン
  base.css           リセットCSSと基本タイポグラフィ
  layout.css         ヘッダーやメイン領域などのアプリ全体レイアウト
  components.css     ボタン・チップ・検索窓・ダイアログなど共通部品
  list.css           楽曲一覧（1列/3列/5列）のレイアウト
  detail.css         楽曲詳細画面のレイアウト
  player.css         再生バー・ミニプレイヤー（譜面風シークバーを含む）
  settings.css       設定パネル

js/
  main.js            起動処理（データ取得→初期化→ルーター起動）の司令塔
  state.js           アプリ全体の状態（フィルタ・並び替え・設定など）
  api.js             データ取得と楽曲データの正規化、画像/音声URLの組み立て
  characters.js       キャラクター（ゲーム内/ゲーム外）の名前・アイコン解決
  query.js            絞り込み・並び替え・検索の純粋ロジック
  badges.js           ユニットタグ／難易度バッジの共通HTML生成
  list-view.js        一覧画面の描画と表示形式（1/3/5列）切り替え
  filter.js           絞り込みパネル（ユニット・書き下ろし・歌唱キャラAND検索）
  sort.js             並び替えパネル（項目・昇順降順・基準難易度）
  search.js           検索ボックスとリアルタイム候補表示
  detail-view.js       楽曲詳細画面の描画
  player.js            音声再生・シーク・バックグラウンド再生・冒頭スキップ
  settings.js           設定パネル（冒頭スキップのON/OFF）
  router.js              ハッシュベースの画面遷移（一覧⇄詳細）
  icons.js                アイコン（インラインSVGスプライト、絵文字は不使用）
  utils.js                 汎用ヘルパー（日付整形・確認ダイアログ等）
```

## データソース

- 楽曲・難易度・タグ・ボーカル・オリジナル動画・キャラクター情報:
  `sekai-world/sekai-master-db-diff`（GitHub Pages配信のJSON）
- ジャケット画像・キャラクターアイコン・音声ファイル:
  `storage.sekai.best` のアセットCDN

すべて公開されている非公式データソースを利用しています。取得は実行時（ブラウザ上）に行われ、
ビルド時にデータを埋め込むことはしていません。

## 実装メモ

- 楽曲一覧のメタデータ（JSON）は起動時に一括取得しますが、**音声ファイルは楽曲詳細を開いて
  ボーカルを選択したときにのみ**取得します。自動再生はせず、再生ボタンを押したときのみ再生します。
- 表示形式は1列（タイトルメイン）/3列・5列（サムネイルメイン）を切り替え可能。5列表示は
  画面幅が狭い場合（700px未満）は選択肢自体を非表示にしています。
- 歌唱キャラクターの絞り込みは複数選択のAND検索です（選択した全キャラクターが
  歌唱に含まれる楽曲のみ表示）。
- オリジナルビデオのボタンは押下時に確認ダイアログを挟んでから外部サイトを開きます。
- 設定の「冒頭スキップ」をONにすると、そのブラウザでは次回以降も設定が保持されます
  （localStorageに保存）。
